## 开发准则

一些建议的规范，根据本人多年开发踩过的坑总结的一些经验

### 异常处理

#### 避免将异常抛出到上层标准类

例如下面的代码，如果 execute() 方法内不手动捕获异常，如果异常抛出到上层的标准类，则异常日志将不会打印到集成日志，而是打印到 ecology 日志，打印到 ecology 日志有个弊端，就是文件很大，不利于查找日志，我建议将所有二开日志打印到集成日志，另外在上层类中不确定异常是如何处理，可能不会在日志中打印异常栈，导致错误难以查找。

❌ **没有捕获异常，异常将抛出到上层**

```java
public class InterfaceParamMappingDetailCmd extends AbstractCommand<InterfaceParamMappingDetail> {
    @Override
    public InterfaceParamMappingDetail execute(CommandContext commandContext) {
        if (id == null || id <= 0) {
            return null;
        }
        InterfaceParamMappingDetail detail = mappingDao.queryDetailMain(id);
        if (detail == null || detail.getId() == null) {
            return detail;
        }
        List<InterfaceParamMappingDetail.ParamMappingItem> mappings = buildMappings(detail);
        detail.setMappings(mappings);
        return detail;
    }
}
```

✅ **手动捕获异常并打印**

```java
public class InterfaceParamMappingDetailCmd extends AbstractCommand<InterfaceParamMappingDetail> {
    @Override
    public InterfaceParamMappingDetail execute(CommandContext commandContext) {
        try {
            if (id == null || id <= 0) {
                return null;
            }
            InterfaceParamMappingDetail detail = mappingDao.queryDetailMain(id);
            if (detail == null || detail.getId() == null) {
                return detail;
            }
            List<InterfaceParamMappingDetail.ParamMappingItem> mappings = buildMappings(detail);
            detail.setMappings(mappings);
            return detail;
        } catch (Exception e) {
            log.error("执行发生异常", e);
            throw  e;
        }
    }
}
```

对于其它地方也是如此，比如 WEB 接口类中的方法、流程 Action 类、计划任务等

**WEB 接口类中的方法手动捕获异常：**
（注：WebExceptionHandler.handle() 内已记录异常信息）

```java
@Path("handle")
@POST
@Consumes(MediaType.APPLICATION_JSON)
@Produces(MediaType.APPLICATION_JSON)
public Response handle(@Context HttpServletRequest request,@Context HttpServletResponse response) {
    Map<String, Object> params = ParamUtil.requestJson2Map(request);
    try {
        Map<String, Object> result = getService().handle(params);
        return Response.ok(result).build();
    } catch (Exception e) {
        return WebExceptionHandler.handle(e);
    }
}
```

#### 不要直接打印异常

不要直接使用 `e.printStackTrace()` 打印异常，这样异常不会记录到集成日志，请将异常信息记录到日志

❌ **错误示例：**

```java
try{
    
}catch(Exception e){
    e.printStackTrace();
}
```

✅ **正确示例：**

```java
try{
    
}catch(Exception e){
    // 将异常打印到集成日志
    log.error("执行发生异常",e);
    // 对异常进行其它处理
}
```

### RecordSet 的使用

#### 不要忽略 execute() 方法的执行结果

以及类似的方法：`executeQuery()`、`executeUpdate()` 等。

这些方法的返回结果可以判断 sql 是否执行正确，如果忽略返回结果就不知道 sql 是否执行正确了，如果 sql 执行错误了但没有进行处理，继续执行后续的业务代码将会导致问题影响进一步扩大，难以排查原因。

❌ **不对返回结果进行判断，忽略结果**

```java
recordSet.executeQuery(sql, params.toArray())
```

✅ **判断返回结果，如果 SQL 执行失败则进行处理，例如抛出异常**

```java
if (!recordSet.executeQuery(sql, params.toArray())) {
    throw new SqlExecuteException("执行sql失败，sql: " + sql + "，params: ["
            + formId + ", " + fieldDbName + ", " + detailTableName + "]", sql);
}
```

#### 使用 getString()获取数值类型

`getInt()`、`getDouble()`、`getFloat()` 等方法会将字段值转换为指定的类型，因为返回的是基本类型，如果字段值为空将返回意想不到的值，例如 `getInt()` 将返回 -1，如果你没有对这样情况的返回结果进行判断，则错误的结果将会导致后面的业务出现错误，最好是先使用 `getString()` 获取字符串值，判断是否为空，然后再转换成指定类型。

可以使用 HuTool 中的 Convert 类转换成指定类型，如果转换失败它会返回 null，也就是字段值为空的时候返回 null，这样就知道这个字段是不是空的了

```java
Convert.toInt(recordSet.getString("parent_id"))
```

### 日志处理

1. 统一将二次开发日志输出到集成日志，不要将日志输出到 ecology 日志，ecology 日志很大日志不好找
2. 捕获异常时，将异常打印到日志，这样可以在日志中看到异常栈，不可以只打印异常信息，例如：

```java
try{
    
}catch(Exception e){
    // 将异常打印到集成日志
    log.error("执行发生异常",e);
    // 对异常进行其它处理
}
```

## 数据库操作

### RecordSet 数据库操作类

代码中的数据库操作使用 RecordSet 类执行，使用时需要创建 RecordSet 对象

```java
RecordSet recordSet = new RecordSet();
```

进行查询操作使用 `executeQuery(sql,params)` 方法，该方法返回布尔值，如果执行不成功则返回 false，其中 params 参数是 sql 语句中的占位符(?)参数值，sql 执行时就会将参数的值替换到占位符中，为防止 sql 注入，如无特殊情况都使用占位符进行注入，不得使用字符串连接。

```java
boolean success = recordSet.executeQuery("SELECT ID FROM table WHERE name=? AND age=?",name,age);
if(!success){
    // sql 查询错误
    log.error("sql 查询错误");
}
// 查询操作一定要调用 next 方法，不然获取不到查询结果
recordSet.next();
recordSet.getInt("ID");
// 获取字段时可转换为其它类型
// recordSet.getString("ID");
```

如果需要查询多条数据，则使用 while 循环，`recordSet.next()` 方法可指定下一行查询数据，如果没有下一行则返回 false

```java
recordSet.executeQuery("SELECT ID FROM table WHERE name=? AND age=?",name,age);
while (recordSet.next()) {
    int id = recordSet.getInt("ID");
}
```

数据库修改操作使用 `executeUpdate(sql,params)` 方法执行，update, insert, delete 操作都可以使用此方法，如果执行失败则返回 false

```java
boolean success = recordSet.executeQuery("UPDATE name=?,age=? FROM table WHERE id=?",name,age,id);
if(!success){
    // 更新失败
    log.error("更新失败");
}
```

`execute(sql)` 也能执行 sql 查询和修改，但不建议使用，不能使用占位符，操作类型不明确，不能通过方法名确定是查询还是修改。

RecordSet 对数据库进行操作时不会在当前代码块抛出异常，如果 sql 执行不成功方法会返回 false。

### 字段值获取方式

RecordSet 获取查询的字段值可以有以下方式

**通过字段名获取**，这种是最常用的

```java
recordSet.getString("name");
```

**通过列位置获取**，比如查询有两个字段，第一个字段的列位置就是1，第二个字段的列位置是2，列位置是从1开始的

```java
recordSet.getString(1)
```

### 数字类型字段值获取问题

如果使用 `recordSet.getInt("field")` 或 `recordSet.getDouble("field")` 获取数字类型的字段值，如果字段值为空则获取到的值可能为 -1，在代码中需要注意判断，或者可以先使用 `recordSet.getString("field")` 获取到字符串值，判断是否为空，不为空才转为数字类型。

### RecordSetTrans 数据库操作类（事务）

使用该类可执行事务

**主要方法：**

- `setAutoCommit()` 是否自动提交事务，如要要事务回滚，需要设为false
- `commit()` 提交事务，在执行更新或插入操作后，可提交事务，使更改生效
- `rollback()` 回滚事务，在数据库执行失败（如返回 false 或抛出异常）可执行回滚操作，撤回之前进行的插入或更新。

### RecordSetExecutionInterface 数据库操作接口

RecordSet 和 RecordSetTrans 这些数据库操作类都实现了 RecordSetExecutionInterface 接口，因此在方法中使用 RecordSetExecutionInterface 接口而不是具体的实现类对于事务执行很有帮助，如果不需要执行事务，则传入 RecordSet，如果需要执行事务则传入 RecordSetTrans。

```java
public static boolean insertWithRsInterface(String tableName, Map<String, Object> fieldData,
                                            RecordSetExecutionInterface recordSet) throws Exception {

}
```

**使用方法：**

`executeSql()` 方法可执行sql，可进行查询或修改（插入），可使用占位符注入参数，参数说明：

1. **sql**：执行sql
2. **isQuerySql**：是否为查询sql，如果是查询传入 true，如果是修改（插入）传入 false
3. **poolName**：连接池名称，用于连接外部数据源，如果是本地数据源传入空字符串
4. **hasNewConn**：是否新建连接，传入 true 或 false 都可以
5. **params**：sql 中的参数值，如果使用占位符注入参数（占位符:?）需要传入该参数

**sql 查询示例：**

```java
RecordSetExecutionInterface recordSet = new RecordSet();
if (!recordSet.executeSql(sql, true, "", false, uuid)) {
    logger.error("根据uuid查询数据id失败");
    continue;
}
if (!recordSet.next()) {
    logger.warn("无法查询到数据id，uuid:" + uuid);
    continue;
}
ids.add(recordSet.getInt("id"));
```

**sql 插入示例：**

```java
public static boolean inserByRsInterface(String tableName, Map<String, Object> fieldData,
                                            RecordSetExecutionInterface recordSet) throws Exception {
    String  sql = SqlUtil.buildInsertSql(tableName, new ArrayList<>(fieldData.keySet()));
    List<Object> insertData = new ArrayList<>(fieldData.values());
    logger.info("sql:" + sql);
    return recordSet.executeSql(sql,false,"",false ,insertData.toArray());
}
```

## java配置文件

### 属性获取

#### 使用 BaseBean 获取（不推荐）

需要创建 BaseBean 对象，不推荐。

```java
BaseBean baseBean = new BaseBean();        
String propValue = baseBean.getPropValue("testProp", "sms");
```

#### 使用 Prop 类获取（推荐）

使用 `weaver.file.Prop` 获取配置文件属性，注意如果配置文件属性值有中文，获取属性值时会导致乱码，原因是没有使用 UTF-8 编码。

```java
// fileName：文件名，不需要带文件扩展名，propName：属性名
String value = Prop.getPropValue(fileName, propName);
```

属性值有中文时，只能将中文转为 unicode 放到配置文件中

转换地址：[在线 Unicode 编码转换 | 菜鸟工具](https://c.runoob.com/front-end/3602/)

#### 使用 Properties 类获取（支持中文）

使用 `java.util.Properties` 类获取配置属性可解决中文乱码问题，不过这种用法比较复杂，建议在有中文的情况下使用

首先需要获取 Properties对象

```java
/**
 * 获取配置文件，解决了配置文件中文乱码问题
 *
 * @param propFieldName 配置文件名称，不需要带扩展名后缀
 * @return Properties对象
 */
public static Properties fetchProperties(String propFieldName) {
    Properties prop = new Properties();
    String path = GCONST.getPropertyPath() + propFieldName + ".properties";
    try (InputStream inputStream = Files.newInputStream(Paths.get(path))) {
        prop.load(new InputStreamReader(inputStream, StandardCharsets.UTF_8));
    } catch (IOException e) {
        log.error("获取配置文件失败", e);
    }
    return prop;
}
```

然后就能获取属性值

```java
Properties properties = fetchProperties(fileName);
String value = properties.getProperty(propName);
```

### ImageFileManager 附件类

ImageFileManager 类可获取附件的相关信息

```java
ImageFileManager ifm = new ImageFileManager();

is = ifm.getInputStream();

```

#### 获取附件文件

调用 `ImageFileManager.getInputStreamById(imagefileId)` 可以获取附件文件的输入流，然后通过输入流保存附件，例如

```java
try(InputStream inputStream = ImageFileManager.getInputStreamById(imagefileId)) {
    File file = new File(file_TO);
    // 保存附件到指定位置
    copyInputStreamToFile(inputStream, file);
} catch (IOException e) {
    e.printStackTrace();

```

获取的输入流并不是数据库中的压缩包文件，而是实际附件文件的输入流

## 根据接口路径找到对应的类

开发人员如果要查看标准文件的代码就要根据接口的请求路径找到相应的代码文件，虽然可以根据接口路径按包名和文件名去查找，但有些麻烦而且有时候路径跟包名和文件名是对不上的，以下提供了便捷的方法可以迅速找到接口路径对应的文件

如果有源码则可以通过idea的自带工具找到接口路径对应的类（如下图），但如果接口只存放在jar包里或class文件里就不行了

### 原理

根据类和方法上的注解，获取到指定的类，比如暴露接口的类和方法都有映射接口路径的@Path注解，这样就可以根据注解上的值来找到类了，对于其它映射接口路径注解也是一样的，比如@RequestMapping

### 步骤

前提是要搭好开发环境

使用 reflections 框架根据注解查找类

1. **添加 reflections 框架依赖**

```xml
<dependency>
    <groupId>org.reflections</groupId>
    <artifactId>reflections</artifactId>
    <version>0.9.11</version>
</dependency>
<dependency> 
    <groupId>com.google.guava</groupId>
    <artifactId>guava</artifactId>
    <version>21.0</version>
</dependency>
```

2. **编写查找代码**，可根据接口路径的上级路径和最终路径找到对应的类。

```java
public String getApiFilePath(String packagePrefix, String parentPath,
                             String endPath) throws Exception {
    // 可以在构造方法加上包路径，这样就会扫描指定包下的类，如 com.api ,如不指定包名则会扫描所有类，这样会比较慢
    Reflections f = new Reflections(packagePrefix);
    // 找到所有使用Path注解的类
    Set<Class<?>> set = f.getTypesAnnotatedWith(Path.class);
    for (Class<?> c : set) {
        Path path = c.getAnnotation(Path.class);
        // 该类的Path注解是否包含指定路径
        if (path.value() != null && path.value().contains(parentPath)) {
            System.out.println("找到类："+c.getName());
            // 获取该类的所有方法
            Method[] methods = c.getMethods();
            for (Method m : methods) {
                // 该方法是否使用Path注解，并且该注解是否包含指定接口路径
                if (m.isAnnotationPresent(Path.class)) {
                    Path methodPath = m.getAnnotation(Path.class);
                    if (methodPath.value().contains(endPath)) {
                        return c.getName();
                    }
                }
            }
        }
    }
    return "";
}
```

3. **示例：** 查找 `/api/workflow/reqform/detailData` 接口路径对应的类示例

类的Path注解一般是接口路径的前面几层，比如/workflow/reqform，接口上级路径可以按 reqform 查找，或者 workflow/reqform，

方法的Path注解就是接口路径的最后一层，按 detailData 进行查找

```java
@Test
public void findApiFile() throws Exception {
    String apiFilePath = getApiFilePath("com", "reqform", "detailData");
    if (StringUtils.isNotBlank(apiFilePath)) {
        System.out.println(apiFilePath);
    } else {
        System.out.println("未找到文件");
    }
}
```

输出结果：

```

com.api.workflow.web.RequestFormNewAction
```

### OA 接口认证

如果没有配置接口白名单，需要获取 token 才能调用 OA 接口。

**认证步骤：**

1. **进行注册认证**
   - 对应接口：`/api/ec/dev/auth/regist`
   - 得到secret和spk

2. **获得token**
   - 对应接口：`/api/ec/dev/auth/applytoken`
   - 用上一步获得的secret和spk作为参数，获得token

```java
@Test
public void Test() throws Exception {
    // 进行注册认证，获取secret和spk，根据业务保存这两个数据
    JSONObject registerResult = JSONObject.parseObject(register());
    String secret = registerResult.getString("secret");
    String spk = registerResult.getString("spk");
    // 利用上一步获得的secret和spk，获取token
    String token = getToken(secret,spk);
    System.out.println("token:"+token);
}

public String getToken(String secret,String spk){
    RSA rsa = new RSA();
    String newSecret = rsa.encrypt(null,secret,null,"utf-8",spk,false);
    String result = HttpRequest.post("http://localhost/api/ec/dev/auth/applytoken")
            .header("appid","test")
            .header("secret",newSecret)
            .execute().body();
    JSONObject resultObj = JSONObject.parseObject(result);
    return resultObj.getString("token");
}

public String register(){
    return  HttpRequest.post("http://localhost/api/ec/dev/auth/regist")
            .header("appid","test")
            .execute().body();
}
```

```java
import org.jasig.cas.client.authentication.AttributePrincipal;
import org.jasig.cas.client.validation.Assertion;

import javax.servlet.*;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import javax.servlet.http.HttpSession;
import java.io.IOException;

/**
 * @author 姚礼林
 * @desc TODO
 * @date 2023/6/15
 */
public class LoginFilter implements Filter {
    @Override
    public void init(FilterConfig filterConfig) throws ServletException {

    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) throws IOException, ServletException {
        HttpServletRequest servletRequest = (HttpServletRequest)request;
        HttpServletResponse servletResponse = (HttpServletResponse) response;
        HttpSession session = servletRequest.getSession(false);
        Assertion assertion = session != null ? (Assertion)session.getAttribute("_const_cas_assertion_") : null;
        AttributePrincipal principal = assertion.getPrincipal();
        // 获取到用户名，构造自己的会话完成登录

        if (!"".equals(name)){
            session.setAttribute("userInfo",name);
            System.out.println("userName:"+name);
            chain.doFilter(request, response);
        }else {
            servletResponse.sendRedirect("toLogin");
        }
    }

    @Override
    public void destroy() {

    }
```

### DefaultWeaTable 表格组件

很多前端的表格组件中的数据和样式并非在前端实现，而是在后端使用 AbstractWeaTable的实现类来生成表格数据和样式，比如列名、数据操作等。

DefaultWeaTable是 AbstractWeaTable的实现类，可以用此类来获取前端表格的数据、sql等信息。

关于表格组件SDK说明：[泛微在线文档](https://e-cloudstore.com/doc.html?appId=da5134c3c57f4c6f87815cf1473f674f)

#### 前端表格数据生成原理

在前端表格组件中会请求一个接口来获取表格的 sessionKey，是先在后端中构建表格组件类，也就是AbstractWeaTable的实现类，把数据sql、表格样式等信息写进去，并将此表格组件类与 sessionKey 绑定。

获取 sessionKey 后，前端会调用一个通用接口：`/api/ec/dev/table/datas`，并携带此 sessionKey，来获取表格数据、列名、分页等信息，此接口就是用此 sessionKey 来获取之前的AbstractWeaTable实现类对象来获取表格信息，生成表格数据。

#### 修改表格 SQL

可以在后端做个接口拦截，拦截生成 sessionKey 的接口，获取表格对象，使用该表格对象修改表格的 SQL

可以在前端接口请求时添加标记参数，来在后端判是否是前端指定位置请求的接口。

在后端接口拦截中通过接口返回数据获取到 sessionKey，然后获取表格对象

```java
String sessionKey = result.getString("sessionkey");
DefaultWeaTable table;
try {
    // 根据 sessionKey 获取表格对象
    table = WeaTableTools.checkTableStringConfig(sessionKey, DefaultWeaTable.class);
} catch (Exception e) {
    log.error("获取表格对象发生异常", e);
    return param.getData();
}
```

然后就能获取到表格数据的 sql 语句

```java
String sqlWhere = table.getSqlwhere();
String sqlForm = table.getSqlform();
String backFields = table.getBackfields();
```

表格组件中也提供了 sql setter 方法，可以将修改的 sql 传入 setter 方法中，例如修改 where 条件

```java
table.setSqlwhere(sqlWhere);
```

对表格对象进行修改后，需要调用 `WeaTableTools.setTableStringVal(sessionKey, WeaTableTools.toTableString(table))` 方法，这样才能让修改生效。

## 多语言处理

可以使用标准工具对多语言进行处理，详细可以看标准工具章节。另外也可以使用下面的代码对多语言进行处理。

传入中文名和英文名，转换成多语言格式

```java
public String getMultiLanguage(String cnName,String enName) {
    recordSet.executeQuery("SELECT id,language from syslanguage");
    int chineseId = 0;
    int englishId = 0;
    while (recordSet.next()) {
        if ("简体中文".equals(recordSet.getString("language"))) {
            chineseId = recordSet.getInt("id");
        }
        if ("English".equals(recordSet.getString("language"))) {
            englishId = recordSet.getInt("id");
        }
    }
    StringBuilder builder = new StringBuilder("~`");
    builder.append(String.format("~`%d %s",chineseId, cnName));
    if (!enName.isEmpty()) {
        builder.append(String.format("~`%d %s",englishId, enName));
    }
    builder.append("`~");
    return builder.toString();
}
```

## 开发问题

### 系统环境

#### 读取文件异常 java.nio.file.InvalidPathException: Malformed input or input contains unmappable characters

执行以下代码会出现该异常，这是由于Paths 类无法解析文件路径，当文件路径包含中文时就会出现异常，这可能和 jvm 的编码配置有关

```java
Files.exists(Paths.get(imageFilePath))
```

**解决方法：**

检查 jvm 编码配置，访问 `[系统地址]/jsp/test1.jsp`

访问页面后如果输出的 `sun.jnu.encoding` 不是 UTF-8 或者 GBK，则表示 jvm 编码配置不正确

需要修改 `/weaver/Resin/conf/resin.properties` 文件，在`jvm_args` 属性最后面添加 `-Dsun.jnu.encoding=UTF-8`，然后重启。

#### 提示出错，但是日志找不到报错

一般对于 Error 级别的异常（比如 ClassNotFound、NoSuchMethod 异常）都可以在 ecology 日志和 Resin 日志中找到，对于其它运行时异常，你必须在代码中进行捕获，让它打印到集成日志中。
